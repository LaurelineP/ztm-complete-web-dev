Asynchronous

Asynchronous

JS is able to communicate with server to request data or share them. The communication could take some times and JS should be able to do them without having to wait for them in order to have a fluid navigation for users and not to block its use and the code execution: this is asynchronous concept

HTTP/HTTPS

HTTP and HTTPS are protocols that allows to communicate with
another machine: another server to transfer some payload or information.
A payload is a a set of data that you want to get, modify or send.

HTTP/HTTPS Methods

In order to instruct the protocol what to do, the developer needs to specify what kind of
"actions" the request is doing. The request presents a "request" and a "response". The request is sent by the client ( browser website ) to the server and the response is sent by the server.

  • GET: to get data from the server
  • POST: to send data to the server
  • PUT: to edit data in the server
  • DELETE: to delete data in the server
  • PATCH: to delete data in the server


Server's response

The server respond with potentially 2 elements: a status code and may contain a payload of any kind ( data, files, etc... )

  • 1xx: information
  • 2xx: successful
  • 3xx: redirection
  • 4xx: client errors
  • 5xx: server error

JSON

There are few ways of sending text data through requests and JSON nowadays standard way to do so. A previous way to send data is XML ( which is more verbose and with "tag" way)

  • JSON: JavaScript Object Notation
  • being a standard to store and exchanging data
  • being text written in key/value pair with fields being string

Handle JSON

  • JSON.stringify(<object>): from a JS Object, outputs the JSON version
  • JSON.parse(<object>): from a JSON value, outputs the JS Object from it

Ajax

Ajax allows to update a part of web page instead of updating the whole content each time an interaction occurred - it changes a part of a content dynamically. The different ways to requests over times relies on ajax.

  • Ajax (JS API) - Uses an XHTMLRequest (XHR - old way to make request )
  • Jquery (third part library) - leaner syntax on ajax ( old way )
  • fetch ( JS Browser API - using ajax under the hood nowadays old way )

Handle JSON

  • JSON.stringify(<object>): from a JS Object, outputs the JSON version
  • JSON.parse(<object>): from a JSON value, outputs the JS Object from it

Promises

Promise is a new feature from ES6: ensuring you to do a request and always get back to you ( later ) with some server response ( a resolved or a rejected response ). It uses callback to resolve the response

Example


const promiseResult = new Promise(( resolve, reject) => {
  if( true ){ return resolve('promise successful')}
  return reject('promise error')
})

promiseResult
  .then( response => response )
  .then( response2 => {
    console.log('response 2:', response2 )
  })
  .catch( error => console.error( {error} ))

Handle JSON

  • JSON.stringify(<object>): from a JS Object, outputs the JSON version
  • JSON.parse(<object>): from a JSON value, outputs the JS Object from it

Promises and Methods

ESNext allowed to make requests ( using XHTMLRequest ) in a cleaner way (e.g.: XHTMLRequest > Ajax > promise with "then" and "catch"> async await ) But ESNext did also provide extra proposition to handle promises with new methods

  • Promise.all(): allows to handle multiple promises to output one promise to process
    • all promises must be successful in order for promise.all to resolve the async requests
    • if one is invalid, all the promise resolutions fail too
  • Promise.allSettled(): allows to handle multiple promises in one result
    • some promise could fail, ".allSettled" will still be able to resolve the one that do not fail and process the one failing
  • Promise.any(): returns first fulfilled request
