    How to apply the component concept to an HTML project? Generally shared elements are within a shared/ folder
    Those shared files should be imported using a nested browsing context HTML element (iframe, embed)
    in order to reuse the same code (one piece of code for all pages) The current navigation list is the same code but use in all pages using an iframe

    • How to structure with a modular approach:

      1. create a shared folder
      2. add whatever reusable html content that can be reused (ex: the navigation)
      3. the shared file(s) should only contain the elements necessary*
        • (no need to add !DOCTYPE html, or html, or head, or body tags )
        • if anchor links are part of the content:
          - in the attribute src use relative path(s) relative to your shared folder
             for each page to be able to redirect properly to the link
          - add target attribute to _top or _self
      4. Use the iframe tag to display the HTML file
      5. Add to iframe tag the attribute src with relative path(s) of your source (A shared file here)
    Could we nest header tags within li tags?

    Yes, list elements (li) can include headers tags along few other tags

    iframe or embed tag?

    iframe or embed are element that allow to display files, media from other sources, contextualized in the element The contextualization mentioned refers to being able to functionally interact with the content from the iframe contextualized in the current webpage even if the source of the iframe or embed tag are external

    Prefer using iframe as embed is deprecated.

    When to use button or input tags?

    button can accept child elements whereas input does not
    so if you need to customized a clickable element ( like adding an icon ) you may
    prefer using button instead of an input.
    However an input has more attached insights and events handling: if one want to have
    a maximum control over the input: better use the input instead

    Prefer using iframe as embed is deprecated.

    When to choose canvas or iframe ?

    canvas is used to depict graphical element and the need to draw.
    iframe is used to offer a preview with a context of a content, webpage and such.
    the iframe context is the ability to interact with the iframe content as if it was part of the
    current page.
