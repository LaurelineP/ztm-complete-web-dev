DOM Manipulation

DOM (Document Object Model) is the ability
to manipulate how what the page content is displaying while this one is already loaded
and being used. This capacity require is due to both HTML content and JavaScript:
JavaScript is the one allowing to change element in the HTML instead of getting another HTML.
This precede nowadays usage of javascript: before jquery, components and virtual DOM.

DOM Selectors and Changing Styles

Selectors are ways to access the HTML content such as the elements, ids, classes ...
Those are the same syntax seen in CSS elements.

  • class are accessible with the name of the class prepended by a dot: .myClass
  • id are accessible with the name of the id prepended by a hash: #myId

DOM Selectors

document.getElementsByTagName(<tag-element>)
Allows to access all the specified tag in the page.

document.getElementsByClassName(<class-name>)
Allows to access all the specified elements with the class in the page.

document.getElementsById(<id-name>)
Allows to access all the specified element with the id in the page.

document.querySelector(<selector>)
Allows to access the first element with the specified selector in the page.

document.querySelectorAll(<selector>)
Allows to access all the elements with the specified selector in the page.

<element>.getAttribute(<attribute-name>)
Allows to access the specified attribute(s) from to the bound element.

<element>.setAttribute(<attribute-name>)
Allows to set an attribute's value to the bound element.

Once one element or more have been selected, it is possible to modify their styles
using the element's following properties

Changing styles

<element>.style = <style-in-string>
Allows to replace or provide a style to the bound element

<element>.class = <class-name>
Allows to replace or provide a class to the bound element,
this will allow to define the matching CSS rule.

<element>.classList
Allows to get the list of all the classes the element has,
this will allow to use the methods attached to it ( add, remove, toggle, ... ).

  • <element>.classList.add( <class-name> )
    Allows to add a class name to the existing empty or non-empty list of class.
  • <element>.classList.remove( <class-name> )
    Allows to remove a class name from the existing empty or non-empty list of class.
  • <element>.classList.toggle( <class-name> )
    Allows to add/remove ( toggling ) a class name from the existing empty or non-empty list of class.

Changing HTML text element's content

<element>.innerHTML = <text-or-tag-in-string>
Allows to replace / override the text content within the text element / tag.

Accessing relative element to a selected element

<child-element>.parentElement
Allows to access the parent element ( common above element ) to the selected element.

<parent-element>.children
Allows to access to all elements within the parent's element,

DOM Events

DOM Events represents actions/interactions triggered by the user on a webpage.
Those correspond to an element property key that accept a function as value.
Those function value are referred to "handlers" or "function handlers":
functions handling an event,

Types of events

  • with the mouse
  • with the keyboard
  • and more...

See DOM Events references documentation

Events Listeners

All of the following methods are HTML element's methods that allow, in a declarative way,
to attach a listener taking a function has parameter.

A function passed to another function is called a callback.

This very same callback in this event listener circumstances, could be called an
handler function: function handling the logic to process on an event fired/triggered.


  • <element>.addEventListener(<event-name>,<function>):
    adds/attaches a function to an element's event.
  • <element>.removeEventListener(<event-name><function>):
    removes a specific function from an element's event.

Mouse Events

Mouse Events are events dispatched when a user interact with her/his mouse.
Each event, identified by its name, has its own purpose

  • click:
    event triggered on clicking on an element using the mouse.
  • dblclick:
    event triggered on double clicking an element using the mouse.
  • mouseover:
    event triggered on mouse is hovering an element.
  • mouseenter:
    event triggered on the cursor is entering the element box-content.
  • mousemove:
    event triggered on the cursor moving within the element box-content.
  • mouseleave:
    event triggered on cursor leaving the element box-content.
  • scroll:
    event triggered on cursor scrolling within an element box-content.

Keyboard Events

Keyboard Events are events dispatched when a user interact with her/his keyboard.
Each event, identified by its name, has its own purpose

  • change:
    event triggered when the element field value did change.
  • input:
    event triggered when the input element value did change.
  • submit:
    event triggered when the form element values did change.
  • keypress:
    event triggered any key of the keyboard is pressed.
