Brad Frost - Atomic Design

  • Organism: a combination of multiple atoms and/or atoms together

One way data binding

React is a whole ecosystem putting in relations components and data - being aware of the app state based on the user experience.

Like mentioned above, React connects components together acknowledging the relations between them, the user interactions and data.

This will allow to keep tracks on how component should be displayed - Component can be reused in different circumstances which customizes the component content.

The Virtual DOM

The Virtual DOM is a representation of the existing rendered DOM as an object.

When a change occurred, the Virtual DOM allows to only affects the elements concerns ( = doing an optimized DOM manipulation for your ) , all the other elements that do not need to be updated will not render; which increases performances. This could also be represented by the above representation.

Setup

  • Resources

  • React Installation recap
    • npm install react:
      would be the way to manually config react - not covered
    • npx create-react-app <app-name>:
      would be the covered way
  • Generated folder and files
    • package.json:
      a file listing all app config
      • app, name, description, author, license and version
      • scripts: to run the app
      • dependencies: packages imported and necessary for the project
      • dev-dependencies packages imported for the project and developers
    • package-lock.json:
      ensuring the app will run anywhere with the resolves internal dependencies of each packages and their versions
    • README.md:
      file generally describing what is the project and how it works
    • .gitignore:
      listing all files and folders to not track in commits hence the repository
    • node_modules:
      all the packages that the app need ( includes all dependencies of all the packages installed in the project )
    • public folder:
      contains files that does not require to be in used in the code app
      • favicon.ico: icon on the browser tab
      • manifest.json: file for your web app to be downloaded if requested to
      • index.html:
        most important file as it is the receptor for your react app to run thanks to the div with the id "root"
