HTML Basics

Header tags:

Header tags are text element allowing to organize the text by a hierarchy.
Headers represent the hierarchy of title and subtitles
They are are going from h1 to h6 going Generally
from the most important to the less important text title

  • Syntax: <h1>...</h1>

  • Example:

    h1 Title

    h2 Title

    h3 Title

    h4 Title

    h5 Title
    h6 Title
Paragraph tag:

Paragraph is a text element allowing to semantically add a text in a HTML document file.
Indeed, text not wrapped by a text element will still display however this is not
a good practice as this will not help the SEO

  • Syntax: <p> ... </p>
  • Example:

    This is a paragraph text

Lists:

There are to different kind of HTML list: unordered list ( ul ) and ordered list ( ol ).
Unordered list is a list that have bullets for list items while unordered list as characters to enumerate each items.
Lists doe come with specific children elements being the list item li
The lists could be nested within another list item creating a listed tree

  • Syntax for ul : < ul > <li> .... <li> </ ul >

    Example : Unordered list

    • list item 1
    • list item 2
    • list item 3
    Note: the above are dotted list already nested
    as it is used to explain this content
    (this is why those are circles and not filled dot)
  • Syntax for ol : < ol > <li> .... <li> </ ol >
  • Example:

    Ordered list

    1. list item 1
    2. list item 2
    3. list item 3

Buttons :

Buttons are interactive HTML elements that the user could trigger by
clicking on them.
There are today, two ways to create a button:

  • using the button tags
  • using the input tag
  • Syntax for button :

    < button
      onclick="console.log('Hello world from button')"
      >
       Click me as a button
      < /button >

    Example :
  • Syntax for input :

    < input
      type="button"
      onclick="console.log('Hello world from input')""
      value="Click me as an input"

HTML Form

Here we will discuss about forms, those bunch of input that one may come across filling
for instance a profile account, or to register to a site and many other use cases.
form is an element that wrap multiple inputs of any kind to let the
user provide information
input s are powerful and are polyvalent with the various type possibilities
HTML provides.
Form and input works together toward the same purpose: handling all data the user provides
at once thanks to connected `attributes` ( name, for, value )

Form:

Syntax form: < form action="" method="" > ... inputs ...< /form >

Syntax input: < input type="" onchange="" for="" name="" placeholder="" value="" >

  • attribute type: allow to handle it type
    (radio, checkbox, date, color, submit, reset ...)
  • specific to form's inputs:
    • label tag: label tag is used to put an insight/description of the input next to it
    • for: allows to bind the value to the context
    • name: allows to bind the value to a for named element
    • value: the user value
