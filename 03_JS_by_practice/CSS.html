CSS basics

Here we will discuss about commonly CSS .
CSS is a bit like the skin on a body/character, it allows to display custom visual aspect
for the current backbone that here may be associated to HTML.

CSS Syntax:

  • A CSS selector: CSS will need to target specific(s) element(s) in the DOM to be able to apply some style
  • A CSS rule: is what is the entire code between curly braces - this defines how the selector will be displayed
  • A CSS property and its value: is always bound to a value ( making a pair of property/value ).
    The property is within the curly braces always preceding by a colon followed by a value
    and semi colon to end the property definition;
<SELECTOR> {
  <PROPERTY>: <VALUE>;
  <PROPERTY>: <VALUE>;
  <PROPERTY>: <VALUE>;
} ...

CSS selectors and properties:

"Selectable" elements can be:

Description CSS syntax
HTML tags elements: <TAG-NAME>
HTML id="" attribute in a tag ): #<ID-NAME>
HTML class="" attribute in a tag ): .<CLASS-NAME>
HTML any attribute <ATTRIBUTE>="<CLASS-NAME>" [<ATTR>="<VALUE>"]
[<ATTR>~="<VALUE>"]
[<ATTR>^="<VALUE>"]
[<ATTR>$="<VALUE>"]

CSS properties are the key value pair within the curly braces,
that defines how the CSS rules is applied.

CSS Advanced

Few others CSS aspect should be covered such as: critical render path, flexbox, responsive.

  • Critical Render Paths : loading files in HTML process
  • Box Models : loading files in HTML process
  • Flexbox : layout property allowing to organize and order elements
  • Animations: with transitions and transform
  • Responsiveness : how to handle different device screen and adapting its content

Critical Render Paths:

The "Critical Render Path" is the concept behind loading imported source of code.
Loading a file in HTML is a blocking process: meaning that each time the HTML
loads a content ( media, file, fonts, etc... )

Box Models

Resource: Box models represent how each element's layout is organized in the following order:
Google: box models

  • margin : pushes the element from the outside
  • border : wrapping the box with a border
  • padding : pushes elements within
  • content : the element content itself

Note: Box models can be hovered (CSS only)

Flexbox CSS

Flexbox CSS is a box module that allows you to also layout elements in a one dimensional
way ( either in the x or y axis at a time); whereas Grid being a two dimensional organization
that can behave accordingly with the relative space around (space distribution).

Example:
A layout build with Flexbox in a full window width can display differently on a window resized.
Once flexbox has been set, Flexbox have few properties to specify the layout arrangement
with the values offering various possibilities.
It works in combination with other values and need to be considered while setting the next one.

Responsive

"Responsive" defines how the website will be presented on multiple type of devices ( meaning screen size )
Everything feel right, no glitches, no unexpected behaviors in the UI or UX.
In your browser's inspector, there is a dedicated tool in order to verify how a website will behave
on different screen size
cmd + option + j or ctrl + shift + j : to open the inspector tool
device icon: to open the inspector with a specific device mode

  • step 1 - HTML:
    set meta tags viewport in your HTML
    <meta name="viewport" content="width=device-width, initial-scale=1.0" />
  • step 2 - CSS and relative values:
    This could be handled using width and height values along with box models property
    • flexbox box model
    • grid box model
    • viewport values
    • percentage values
